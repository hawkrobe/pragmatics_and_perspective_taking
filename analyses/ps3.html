Part 1: Basic simulation and NHST

Lets start by convincing ourselves that t-tests have the appropriate false positive rate. Run 10,000 t-tests with standard, normally-distributed data from a made up 30-person, single-measurement experiment (the command for sampling from a normal distribution is rnorm). Whats the mean number of significant results?

First do this using a for loop.

sigCount = 0
numSamples = 10000
for(i in 1:numSamples) {
  fakeSample = rnorm(30)
  stat = t.test(fakeSample)
  if(stat$p.value < 0.05) {
    sigCount = sigCount + 1;
  }
}
print(sigCount / numSamples)
## [1] 0.0559

Next, do this using the replicate function:

numSamples = 10000
testRes <- replicate(numSamples, t.test(rnorm(30))$p.value < .05)
cat("false positive rate:", sum(testRes)/numSamples)
## false positive rate: 0.0493

Using both methods, we find a false positive rate of about 0.05! In other words, if the null hypothesis is true and there is no effect, well still report one about 5% of the time. This is our alpha value (in decision-theoretic terms). As the number of samples increases, well converge to this rate exactly.

Ok, that was a bit boring. Lets try something moderately more interesting - lets implement a p-value sniffing simulation, in the style of Simons, Nelson, & Simonsohn (2011).

Consider this scenario: you have done an experiment, again with 30 participants (one observation each, just for simplicity). The question is whether their performance is above chance. You arent going to check the p-value every trial, but lets say you run 30 - then if the p-value is within the range p < .25 and p > .05, you optionally run 30 more and add those data, then test again. But if the original p value is < .05, you call it a day, and if the original is > .25, you also stop.

First, write a function that implements this sampling regime.

double.sample <- function (maxP) {
  fakeSample1 = rnorm(30)
  test1p = t.test(fakeSample1)$p.value
  if(test1p >= maxP || test1p < .05) {
    return (test1p < 0.05)
  } else {
    fakeSample2 = rnorm(30)
    test2p = t.test(c(fakeSample1, fakeSample2))$p.value
    return (test2p < 0.05)
  }
}

Now call this function 10k times and find out what happens.

numSamples = 10000
sum(replicate(numSamples, double.sample(.25)))/numSamples
## [1] 0.0723

Is there an inflation of false positives? How bad is it?

There is indeed an inflation of false positives: we find false positives about 7% of the times instead of 5% as before

Now modify this code so that you can investigate this double the sample rule in a bit more depth. Lets see what happens when you double the sample ANY time p > .05 (not just when p < .25), or when you do it only if p < .5 or < .75. How do these choices affect the false positive rate?

Thought question: Take a moment to interpret the coefficients of the model. Q1 - What are the units? The coefficient in front of dose is the average slope of the line if the two supplement groups were collapsed together. The coefficients in front of the dose groups are the numbers added to this baseline dose at each level to get their respective group means. Q2 - How does the interaction relate to the plot? In the plot, we saw that the slope across doses is steeper for VC supplement than OJ supplement. This is the interaction our model is picking up on. Q3 - Should there be an interaction in the model?
